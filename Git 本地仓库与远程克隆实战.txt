引言
	git是一个开源的分布式版本控制系统 用于记录文件的修改历史，方便多人协作开发和项目管理。它可以让你追踪代码的每一次变动、快速切换不同版本、合并团队成员的工作，同时支持离线操作，因为每个本地仓库都是完整的版本库。

	Windows系统 git的下载 
		打开 Git 官方网站：https://git-scm.com/
		 这里简单讲解linux系统安装git
		Linux 系统 Git 安装

在 Linux 系统中，Git 通常可以通过包管理器快速安装：

Ubuntu / Debian 系列：

sudo apt update
sudo apt install git

CentOS / RHEL 系列：

sudo yum install git

安装完成后，输入：

git --version

确认安装成功即可。

注：不同 Linux 发行版命令略有差异，若使用其他系统，请参考官方文档。
	
	git安装篇结束


创建/初始化本地仓库（最简单）

目标：让目录变成一个可被 Git 管理的仓库。

命令：git init 
效果：目录下生成 .git 文件夹，开始记录版本历史。




添加文件到暂存区
目标：告诉 Git 哪些文件需要记录在下一次提交。

命令：
git add <文件名>
# 或添加当前目录下所有文件
git add .
示例：

echo "# 我的项目" > README.md
git add README.md
简单解释：先修改，再用 git add 暂存。



提交到本地仓库

目标：将暂存区的内容生成版本记录。

命令：
git commit -m "初始化项目结构，添加 README"

解释：

每次提交都是一次“快照”，可以回退和对比。

提交信息要简明描述修改内容。

查看提交历史
命令：

git log

输出：

commit abc123
Author: Your Name <you@example.com>
Date:   2026-03-17

    初始化项目结构，添加 README

作用：查看本地仓库的每次提交历史。



git 基础篇结束


关联远程仓库

目标：把本地仓库和远程仓库建立连接。

命令：

git remote add origin <仓库地址>
git remote -v   # 查看远程仓库
推送到远程仓库（最终进阶）

目标：把本地修改同步到远程，让团队成员可见。

命令：

git push -u origin main

解释：

-u 参数：本地分支和远程分支建立关联，下次可直接 git push。

推送后，远程仓库就有最新版本。


操作流程图
创建本地仓库 → 添加文件 → 提交 → 查看历史 → 关联远程 → 推送
