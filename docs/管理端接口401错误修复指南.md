## 常见问题及解决方案

### 问题1: Token未存储

**症状**: `diagnoseAuth()` 显示 `hasToken: false`

**解决方案**:
1. 检查登录流程是否正确
2. 查看 [src/stores/user.ts:17](file:///d:/Stday/个人简历/项目/我的博客/src/stores/user.ts#L17) 是否正确设置token
3. 确认 `persist: true` 配置生效

**验证**:
```javascript
// 登录后检查
localStorage.getItem('pinia')
// 应该看到包含token的数据
```

### 问题2: Token格式错误

**症状**: Token长度异常或格式不正确

**解决方案**:
1. 检查后端返回的token格式
2. 确认前端正确接收和存储token
3. 查看 [src/api/auth.ts:6-10](file:///d:/Stday/个人简历/项目/我的博客/src/api/auth.ts#L6-L10) 的登录处理

### 问题3: Token过期

**症状**: 之前可以访问，现在返回401

**解决方案**:
1. 清除认证数据重新登录：
   ```javascript
   clearAuthData()
   // 然后重新登录
   ```
2. 考虑实现token自动刷新机制

### 问题4: 后端JWT配置问题

**症状**: Token存在但仍然401

**解决方案**:
1. 检查后端JWT密钥配置
2. 确认token过期时间设置
3. 查看后端日志中的认证错误信息

## 预防措施

### 1. 添加Token刷新机制

建议实现token自动刷新，避免token过期：

```typescript
// 在request.ts中添加
instance.interceptors.response.use(
  (response) => response,
  async (error) => {
    if (error.response?.status === 401) {
      // 尝试刷新token
      const refreshed = await refreshToken()
      if (refreshed) {
        // 重试原请求
        return instance.request(error.config)
      }
    }
    return Promise.reject(error)
  }
)
```
